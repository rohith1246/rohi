# Rohi - Breaking Bad Habits & Recovery Garden

Rohi is a GenAI-powered web application designed to help users break bad habits and recover from addictions. Built on cognitive behavioral therapy (CBT) principles, Rohi focuses on positive reinforcement, replacing punitive streak counters with a virtual **Recovery Garden** that grows over time.

---

## Calming Dark Theme & Accessibility
- **Theme**: Slate Green & Midnight Forest Green palette, optimized for focus and tranquility.
- **a11y**: High-contrast ratios, semantic tags, and keyboard-accessible forms conforming to WCAG AA guidelines.

---

## Core Features
1. **🌳 Virtual Recovery Garden**: Every successful day grows a custom SVG tree through 6 stages (Seed ➔ Sprout ➔ Sapling ➔ Young Tree ➔ Mature Tree ➔ Blooming Tree). Relapses/slips pause the growth and stop the animation, but **never reset your progress to zero**.
2. **💬 Adaptive AI Coach**: CBT-based support therapist (utilizing Gemini with a REST fallback to Groq) that remembers past sessions and dynamically shifts its coaching tone (empathetic, nurturing, motivational, or structured) based on your logged emotional state.
3. **🧠 Proactive AI Nudges**: Analyzes your logged triggers and alerts you on your dashboard to daily pattern warning signs, suggesting concrete CBT alternatives.
4. **🚨 Urge Surfing Emergency Interventions**:
   - Paced Box Breathing widget (Inhale ➔ Hold ➔ Exhale ➔ Hold) to regulate physiology during cravings.
   - On-demand, custom 3-step action plan generated by AI based on your immediate craving context.

---

## Technology Stack
- **Backend**: Flask (Python 3.11), SQLAlchemy ORM.
- **Database**: Neon PostgreSQL / Render PostgreSQL.
- **AI Engine**: Gemini API (`gemini-2.5-flash`), Groq REST API fallback (`llama-3.3-70b-specdec`).
- **Server**: Gunicorn (configured with a 120s timeout in `gunicorn.conf.py` to prevent LLM worker kills).

## Local Setup

### 1. Clone & Enter Project Directory
```bash
git clone https://github.com/rohith1246/rohi.git
cd rohi
```

### 2. Configure Environment Variables
Create a `.env` file in the project root:
```env
DATABASE_URL=postgresql://user:password@host/dbname?sslmode=require
GEMINI_API_KEY=your_gemini_api_key
GROQ_API_KEY=your_groq_api_key_fallback
FLASK_SECRET_KEY=your_random_flask_secret_key
FLASK_DEBUG=True
```

### 3. Run the App
On Windows:
```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

On macOS/Linux:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python3 app.py
```
Open `http://127.0.0.1:5000` in your browser.

## Deploying to Render

To deploy this application to [Render](https://render.com):

1. **New Web Service**: Connect your GitHub repository.
2. **Service Properties**:
   - **Root Directory**: `habit_breaker` (or leave blank if it's at the repository root)
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app` (Render automatically uses our `gunicorn.conf.py` which sets timeouts and worker limits)
3. **Environment Variables**: Add your `DATABASE_URL`, `GEMINI_API_KEY`, `GROQ_API_KEY`, and `FLASK_SECRET_KEY` variables in Render's Environment tab.

The system will build using our committed `.python-version` file to ensure the stable **Python 3.11.8** runtime is utilized.
